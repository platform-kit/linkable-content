# Philosophy

PlatformKit is built on a few core principles:

## 1. **Simplicity First**

We believe in making site-building as simple as possible. Every feature is designed to minimize complexity, reduce boilerplate, and avoid unnecessary abstraction. The default experience should "just work" — no config required.

## 2. **Local-first, Git-powered**

Your content lives in local files, versioned with Git. The CMS is a thin layer on top, never locking you in. You can edit markdown, YAML, or JSON directly, or use the admin UI — your choice.

## 3. **Composable & Modular**

Collections, themes, and overrides are all modular. You can mix and match, extend, or replace any part. The theme system encourages experimentation without breaking core functionality.

## 4. **Open, Transparent, Inspectable**

Everything is open source, readable, and hackable. The build pipeline, config system, and data model are all designed to be inspectable and auditable. No hidden magic.

## 5. **Performance & Accessibility**

Sites are statically built for speed. The default theme is mobile-first, keyboard-accessible, and optimized for search engines and social sharing.

## 6. **Respect for User Data**

No tracking, no analytics by default. When enabled, analytics are privacy-first, using fire-and-forget events and no personal data. Newsletter and subscriber data are always under your control.

## 7. **Future-proof by Design**

Schema migrations, build hooks, and modular config ensure your site can evolve without breaking. Upgrades are safe, and old content is auto-migrated.

---

PlatformKit is for creators who want control, transparency, and simplicity. We aim to empower you to build beautiful, performant sites — your way.
